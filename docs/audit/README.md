# 📂 docs/audit/

Каталог для аудитов, ориентированных на конкретные техники и требования безопасности/качества (маскирование данных, утечки, согласованность с регламентами).

## Назначение
Сюда складываются документы, фиксирующие план и статус реализации технических аудитов: набор правил, тест-кейсы, метрики, чек-лист реализации. Аудиты в этом каталоге, как правило, выполняются один раз в рамках конкретной техники и продолжают жить как реестр требований к этой технике.

> Для широких аналитических ревью архитектуры/процессов используйте [`docs/analysis/`](../analysis/) с типом `audit` в имени файла по [`naming-convention.md`](../standards/naming-convention.md).

## Содержание
- [`data-masking_v1.md`](data-masking_v1.md) — аудит маскирования чувствительных данных при передаче в зарубежные LLM-API.
- [`2026-05-12_repository-consistency_audit_v1.md`](2026-05-12_repository-consistency_audit_v1.md) — аудит согласованности репозитория, полноты документации и тестируемости требований (issue #21).

## Правила
- Файлы версионируются суффиксом `_vN`, исторические версии не удаляются.
- Каждый документ содержит: scope, набор правил/паттернов, тест-кейсы, метрики, статус реализации, открытые вопросы и историю изменений.
- При значимых изменениях создаётся новая версия документа (`_v2.md`), а в шапке указывается ссылка на предыдущую.
