# 2. Кнопки и селекторы

> 📘 **Справка для БА** · файл 2 из 4
> ← [1. Режимы работы](01_intro_modes.md) · Далее → [3. Результаты и экспорт](03_results_and_export.md)

Здесь — карта интерфейса. Каждый элемент описан с точки зрения **«на что
нажимать и что произойдёт»**. Технических деталей нет — только действия и
ожидаемый результат.

---

## 🗂 Общая структура экрана

![Главный экран в режиме «Анализ ТЗ»](screenshots/01_analysis_tab.png)

Интерфейс делится на две зоны:

| Зона | Где расположена | Что делает |
|------|-----------------|------------|
| **Сайдбар (левая панель)** | Слева | Переключение режимов, настройки, очистка истории |
| **Рабочая область** | По центру | Загрузка файла / поле для вопроса, ответы, отчёт |

---

## 1. Сайдбар: переключатель режимов

**Что это.** Радио-кнопки (или вкладки) в левой панели под заголовком
**«Настройки»** → **«Режим работы»**.

**Доступные значения:**

- **📊 Анализ ТЗ** — массовая проверка файла.
- **💬 Консультация по документации** — диалог с базой знаний.

**Что произойдёт при нажатии:** интерфейс мгновенно переключится на нужный
режим. **История диалога будет очищена**, если вы переключаетесь между
режимами (см. [файл 1](01_intro_modes.md)).

> ⚠️ **Внимание.** Если вы вели важный диалог в «Консультации» — сначала
> сохраните его через **«📥 Сохранить диалог (.md)»**, и только потом
> переключайтесь.

---

## 2. Зона загрузки файла (режим «📊 Анализ ТЗ»)

**Что это.** Прямоугольная область **«📎 Файл тендерного ТЗ»** с подсказкой
«Drag and drop file here / Upload».

**Как пользоваться:**

- **Перетащите файл** мышкой прямо в прямоугольник, **или**
- нажмите кнопку **«Upload»** и выберите файл через стандартный диалог
  открытия.

**Какие файлы принимаются:**

| Формат | Поддержка | Комментарий |
|--------|:---------:|-------------|
| `.xlsx` (Excel) | ✅ | Лучший выбор для табличных ТЗ |
| `.docx` (Word) | ✅ | Поддерживается. Структура распознаётся автоматически |
| `.doc` (старый Word) | ❌ | Не поддерживается. Откройте в Word и пересохраните как `.docx` |
| `.pdf`, `.txt`, изображения | ❌ | Не поддерживаются |

**Лимит размера:** **10 МБ** на файл.

**Что произойдёт после загрузки:** имя файла появится под зоной загрузки.
Файл **ещё не обрабатывается** — анализ запустится только после нажатия
«Запустить анализ».

> 💡 **Если файл не принимается:** проверьте расширение и размер. Если файл
> в `.doc` — откройте его в Word и пересохраните как `.docx`.

## 3. Селекторы экспорта

Появляются в режиме «📊 Анализ ТЗ» после загрузки файла.

### 3.1. Формат отчёта

**Что это.** Радио-селектор **«Формат отчета»** с тремя вариантами:

| Значение | Когда выбирать |
|----------|----------------|
| **`.xlsx`** | Хотите дальше работать в Excel — фильтровать, сортировать, считать |
| **`.docx`** | Готовите печатный отчёт или письмо заказчику |
| **`.md`** | Кладёте отчёт в git / Confluence / wiki, нужен plain-text |

**По умолчанию** выбран тот же формат, что и у загруженного файла
(`.xlsx` → `.xlsx`, `.docx` → `.docx`). Можно изменить вручную.

### 3.2. Режим сохранения

**Что это.** Подпись **«Режим сохранения: create_new»** под селектором формата.

**Что означает.** Система **всегда создаёт новый файл** —
исходное ТЗ не модифицируется. Имя итогового файла:
`<имя_исходника>_report_<id8символов>.<расширение>`. Например:
`tender_2026_report_a3b1c9d0.xlsx`.

> 🔒 **Почему так.** Чтобы исходный файл заказчика никогда не пострадал
> от ошибки или сбоя. Версия с пометками — всегда отдельный файл.

---

## 4. Кнопка «Запустить анализ»

**Что это.** Большая красная кнопка **«▶️ Запустить анализ»** под селекторами.

**Что произойдёт при нажатии:**

1. На экране появится индикатор прогресса.
2. Система последовательно обработает каждое требование из файла.
3. Появится счётчик в реальном времени: **«Успешно: X / Ошибки: Y»**.
4. Когда все строки обработаны — снизу появится кнопка
   **«📥 Скачать отчёт»**.

**Сколько ждать.** Ориентир — **до 15 минут на 50 требований** (зависит
от загрузки провайдера LLM).

> ⏳ **Не закрывайте вкладку браузера** во время анализа — прогресс
> сбросится, и придётся запускать заново.

---

## 5. Поле ввода в режиме «💬 Консультация»

**Что это.** Поле **«Задайте вопрос по документации…»** внизу страницы (как
в обычном чате).

**Как пользоваться:**

- Напечатайте вопрос.
- Нажмите **Enter** (или кнопку отправки).
- Дождитесь ответа — обычно несколько секунд.
- Задайте уточняющий вопрос — система помнит контекст до 6 сообщений назад.

**Что появится после отправки:**

- Ваш вопрос — в чате как сообщение пользователя.
- Ответ модели — с цитатами и ссылками на источники.
- Раскрывающийся блок **«Источники»** — фрагменты документации, на которых
  основан ответ.

---

## 6. Кнопка «🧹 Очистить историю»

**Что это.** Кнопка в сайдбаре, **активна только в режиме «Консультация»**.

**Что произойдёт при нажатии:**

- Все сообщения текущего диалога удалятся.
- Появится уведомление **«🧹 История консультации очищена»**.
- Можно начинать новый диалог с чистого листа.

**Когда нажимать:** когда тема разговора изменилась и вы не хотите, чтобы
старые сообщения влияли на новые ответы.

> 💡 **Совет.** Перед очисткой можно **скачать диалог** кнопкой
> **«📥 Сохранить диалог (.md)»**, если он понадобится позже.

---

## 7. Кнопка «Повторить» при ошибке

**Что это.** Кнопка **«Повторить»**, которая появляется **только** под
сообщением об ошибке.

**Что произойдёт при нажатии:** система повторит **тот же самый запрос**,
который не прошёл (не нужно перенабирать вопрос или повторно загружать
файл). Подробнее — в [файле 4: Устранение проблем](04_troubleshooting.md).

---

## 8. Прочие элементы сайдбара

| Элемент | Что делает |
|---------|------------|
| **Версия / Репозиторий / Issues** | Ссылки на код и баг-трекер (для разработчиков) |
| **Статус системы** (UI / Config / Концепция) | Зелёные плашки = всё ок. Жёлтый/красный — сообщите в поддержку |
| **Сколько чанков извлекать** | Сколько фрагментов из базы знаний система покажет. По умолчанию 5 — менять обычно не нужно |
| **Режим отладки** | Для разработчиков. Бизнес-аналитику **не нужен**, оставьте выключенным |
| **`.env` не найден** (предупреждение) | Конфигурация не настроена — обратитесь к администратору |

---

## Шпаргалка: типичный сценарий за 3 клика

**Анализ ТЗ:**

1. **Клик 1:** Загрузить файл (drag-and-drop).
2. **Клик 2:** Нажать **«Запустить анализ»**.
3. **Клик 3:** После завершения — нажать **«📥 Скачать отчёт»**.

**Консультация:**

1. **Клик 1:** Переключить режим на «💬 Консультация» (сбросит историю,
   если была).
2. **Клик 2:** Ввести вопрос и нажать Enter.
3. **Клик 3:** Прочитать ответ. (Опционально — задать уточняющий вопрос.)
