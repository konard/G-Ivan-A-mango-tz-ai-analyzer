# 📘 Концепция внедрения ИИ-анализатора тендерных ТЗ (MVP)
**Версия:** 1.0 | **Дата:** 2024-05-12 | **Статус:** Approved for MVP

---

## 1. Введение
Настоящий документ определяет концепцию внедрения ИИ-инструмента для автоматизированного анализа тендерных технических заданий (ТЗ). Решение применяется на этапе первичного скрининга для классификации требований по категориям: `Да`, `Нет`, `Частично`, `НД` с генерацией обоснований и ссылок на документацию.

**Стандарты:** ISO/IEC 29148 (требования), ISO/IEC 42001 (управление ИИ), ISO/IEC 23894 (риски), NIST AI RMF, BABOK v3.

**Команда и роли:** Product Owner — Ivan Gulienko (@G-Ivan-A), Code Agent — Konstantin Diachenko (@konard), Prompt Owner — Ivan Gulienko (@G-Ivan-A). Подробное описание ролей, ответственностей и матрица RACI — в [`docs/standards/roles.md`](standards/roles.md).

## 2. Бизнес-контекст
| Проблема | Влияние | Метрика успеха (MVP) |
|----------|---------|----------------------|
| Ручной анализ ТЗ (4-8 ч/тендер) | Низкая пропускная способность БА | Время обработки ≤ 90 мин |
| Несогласованность оценок | Риск ошибок в КП | Согласованность с экспертом ≥ 75% (F1) |
| Отсутствие трассируемости | Сложность аудита | 100% обоснований содержат ссылку на источник |

**Ограничения:** Только публичная документация + внутренний перечень функциональности. Маскирование чувствительных данных обязательно. Финальное утверждение остается за БА.

## 3. Описание решения (RAG)
Архитектура: Retrieval-Augmented Generation.
**Компоненты:**
1. Парсер входных файлов (.xlsx/.docx) → извлечение атомарных требований.
2. Векторное хранилище (ChromaDB) + BM25 индекс → гибридный поиск.
3. LLM-классификатор с промпт-контролем → JSON-вывод с цитатами.
4. Экспортер → исходный файл + колонки `[Статус]`, `[Комментарий]`.

**Поток данных:** Файл → Парсинг → RAG-поиск (BM25+Dense+RRF) → LLM → Валидация → Экспорт.

## 4. Требования
**ФТ:** FR-01..FR-08 (парсинг, классификация 4 категорий, обязательное цитирование, экспорт, маскирование, Streamlit UI, логирование сессий).
**НФТ:** Точность ≥75%, Время ≤15 мин/50 требований, Цитируемость ≥95%, Резидентность РФ (для production), Аудируемость сессий.
**Данные:** Чанкинг 200-300 токенов, overlap 50. Модель эмбеддингов: `BAAI/bge-m3`. Реестр источников с хешами.

## 5. Архитектура и стек
**LLM (приоритет fallback):** Qwen (DashScope) → DeepSeek → GigaChat → YandexGPT. Зарубежные API только для тестовых данных (`use_test_data_mode: true`).
**Поиск:** Гибридный (BM25 для точных терминов + Dense для семантики + RRF для ранжирования).
**Промпт-менеджмент:** Версионирование в Git, A/B-тестирование через конфиг, обязательная валидация JSON-схемы.

## 6. Управление рисками
| Риск | Митигация | Мониторинг |
|------|-----------|------------|
| Галлюцинации | Mandatory citation, confidence threshold, BA review flag | Доля валидных цитат ≥90% |
| Устаревание БЗ | Версионирование, хеш-чек файлов, флаг `⚠️` | Задержка индексации ≤24 ч |
| Утечка данных | Regex-маскирование, резидентные провайдеры для production | Аудит логов, 0 утечек |
| Низкое доверие БА | Прозрачный вывод, гайдлайны, быстрая коррекция | Опрос удовлетворенности ≥4.0/5 |

## 7. План внедрения
**MVP (2 нед/16ч):** Парсер Excel → RAG → LLM → Streamlit UI → Тест на 10 требованиях.
**Пилот (3-5 нед):** Подключение 2-3 БА, замер метрик, докрутка промптов, настройка авто-обновления БЗ.
**Масштабирование (6-8 нед):** 100% покрытие продуктов, интеграция с репозиторием документации, передача в поддержку.
**Exit Criteria MVP:** Пайплайн работает, точность ≥70%, UI позволяет загрузить/скачать файл, документация согласована.
